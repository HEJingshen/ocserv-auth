# ocserv-auth
## 概述

`setup-cert-auth.sh` 是一个用于 Linux 系统的 Bash 脚本，自动化配置 **ocserv（OpenConnect VPN 服务器）** 的证书认证体系。它提供交互式菜单，支持证书的创建、续期和吊销操作，并自动修改 `ocserv.conf` 以启用双因素认证（密码 + 证书）。

## 主要功能

| 功能 | 说明 |
|------|------|
| **CA 证书管理** | 自动生成 CA 证书（RSA 4096 位，有效期 ~32 年），检测过期状态，过期时自动重新生成并备份旧 CA |
| **CRL 管理** | 生成并维护证书吊销列表（CRL），用于标记已吊销的用户证书 |
| **用户证书生成** | 为每个用户生成 4 个文件：私钥、证书、标准 `.p12`、iOS 兼容 `.p12`（3DES 加密） |
| **自动续期** | 检测过期或损坏的证书并自动重新生成，跳过有效的证书 |
| **证书吊销** | 通过 `fzf` 交互式多选界面选择用户，吊销后将证书加入 CRL 并删除本地证书文件 |
| **ocserv.conf 自动配置** | 幂等修改配置文件，仅在缺失时插入/取消注释相关配置行，修改前自动备份 |
| **依赖自动安装** | 自动检测并安装 `fzf`、`certtool`（gnutls-bin/gnutls-utils）、`openssl` |
| **OpenSSL 版本兼容** | 自动检测 OpenSSL 主版本，在 3.0+ 上为 iOS `.p12` 文件添加 `-legacy` 标志 |

## 前置要求

### 系统要求

- **Linux 操作系统**
- **root 权限**（必须通过 `sudo` 运行）

### 必需文件

脚本运行前，以下文件必须存在：

| 文件 | 说明 |
|------|------|
| `/etc/ocserv/ocserv.conf` | ocserv 服务器主配置文件 |
| `/etc/ocserv/ocpasswd` | ocserv 密码数据库（格式：`用户名:密码哈希:...`） |

### 依赖工具（脚本自动安装）

- `fzf` — 模糊查找器，用于交互式用户选择
- `certtool` — 来自 `gnutls-bin`（Debian/Ubuntu）或 `gnutls-utils`（CentOS/RHEL）
- `openssl` — 密钥生成和 PKCS#12 导出

## 目录结构

脚本运行后会在 `/etc/ocserv` 下创建以下目录：

```
/etc/ocserv/
├── ocserv.conf              # 主配置文件（脚本会修改）
├── ocpasswd                 # 密码数据库（只读）
├── ca/                      # CA 证书目录
│   ├── ca-cert.pem          # CA 自签名证书
│   ├── ca-key.pem           # CA 私钥（权限 600）
│   ├── crl.pem              # 证书吊销列表
│   └── backup-<timestamp>/  # 旧 CA 备份（重新生成时）
├── user-certs/              # 用户证书目录
│   ├── <username>/
│   │   ├── <username>-key.pem      # 用户私钥（RSA 2048）
│   │   ├── <username>-cert.pem     # 用户证书
│   │   ├── <username>.p12          # 标准 PKCS#12 文件
│   │   └── ios-<username>.p12      # iOS 兼容 PKCS#12（3DES）
│   └── ...
└── config-per-user/         # 每用户配置目录（可选）
```

## 使用方法

### 基本用法

```bash
sudo ./setup-cert-auth.sh
```

脚本启动后会：

1. 检查 root 权限
2. 切换到 `/etc/ocserv` 目录
3. 验证 `ocserv.conf` 和 `ocpasswd` 文件存在
4. 安装缺失的依赖工具
5. 初始化 CA、CRL 和用户列表
6. 显示交互式菜单

### 交互式菜单

```
==============================================
  ocserv 证书认证管理
==============================================

  1) 证书新建 / 续期
  2) 吊销用户证书
  0) 退出

  请选择操作 [0-2]:
```

#### 选项 1：证书新建 / 续期

执行流程：

1. **显示状态表** — 列出所有 `ocpasswd` 中用户的证书状态（有效/已过期/不存在/已吊销/错误）
2. **自动处理** — 对过期、损坏、缺失的证书执行生成/续期，跳过有效证书
3. **配置 ocserv.conf** — 检查 5 项配置并自动补全：
   - `auth = "plain[passwd=/etc/ocserv/ocpasswd]"` — 密码认证
   - `enable-auth = "certificate"` — 证书认证
   - `ca-cert = /etc/ocserv/ca/ca-cert.pem` — CA 证书路径
   - `cert-user-oid = 0.9.2342.19200300.100.1.1` — 用户名 OID
   - `crl = /etc/ocserv/ca/crl.pem` — CRL 路径
4. **创建用户配置目录** — `config-per-user/`（可按用户分配不同网络权限）

状态表示例：

```
  用户             状态         有效期
  ---------------------------------------------------------------
  alice            有效         到期: 2035-08-15 12:00 (3400 天)
  bob              已过期       45 天前过期
  charlie          不存在       需要生成
  david            已吊销       证书已被吊销
```

#### 选项 2：吊销用户证书

1. 显示所有用户及其证书状态
2. 打开 `fzf` 多选界面，支持：
   - `↑↓` 导航
   - `Space` 选择/取消
   - `Tab` 取消全选
   - `Ctrl+A` 全选
   - `Enter` 确认
3. 显示确认列表，输入 `y` 确认
4. 将选中用户的证书加入 CRL，删除本地证书文件

#### 选项 0：退出

退出脚本，不执行任何操作。

## 客户端连接

| 方式 | 命令 |
|------|------|
| **密码登录** | `openconnect <服务器地址>` |
| **证书登录** | `openconnect --certificate user-certs/<用户名>/<用户名>.p12 <服务器地址>` |
| **iOS** | 将 `ios-<用户名>.p12` 文件导入 iPhone，使用 AnyConnect 客户端连接 |

## 使配置生效

修改配置后需重启 ocserv 服务：

```bash
systemctl restart ocserv
```

## 脚本特性

### 幂等性

脚本支持反复运行：

- 有效的证书不会被重新生成
- `ocserv.conf` 仅在配置缺失时才修改
- 重复运行不会产生副作用

### CA 证书过期处理

当检测到 CA 证书过期时：

1. 备份旧 CA 到 `ca/backup-<时间戳>/`
2. 生成新的 CA 证书（新的密钥对和序列号）
3. **注意：** CA 重新生成后，所有现有用户证书将失效，需要重新运行选项 1 为所有用户生成新证书

### 证书参数

| 参数 | 值 |
|------|-----|
| CA 密钥长度 | RSA 4096 位 |
| 用户密钥长度 | RSA 2048 位 |
| CA 有效期 | 11680 天（约 32 年） |
| 用户证书有效期 | 3650 天（10 年） |
| 用户 CN/UID | 用户名 |
| 证书序列号 | 随机 10 字节十六进制（强制正数） |
| CRL 有效期 | 3650 天 |

### 配置文件备份

首次修改 `ocserv.conf` 时：
- 创建 `ocserv.conf.bak` — 静态备份
- 创建 `ocserv.conf.bak.<时间戳>` — 带时间戳的备份

后续运行不再重复备份。

## 注意事项

1. **`ocpasswd` 中的用户名格式** — 仅包含字母、数字、下划线、连字符的用户名会被处理，其他格式会被跳过
2. **吊销操作不可逆** — 吊销后的证书无法恢复，必须为用户重新生成新证书
3. **CA 重建影响所有用户** — 重新生成 CA 后，所有现有用户证书全部失效
4. **iOS 证书兼容性** — iOS `.p12` 文件使用 3DES 加密算法（`-descert`），在 OpenSSL 3.0+ 上自动添加 `-legacy` 标志以兼容旧版 iOS
5. **脚本仅支持 Linux** — 不适用于 macOS 或其他 Unix 系统
